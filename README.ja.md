# music-kanazawa-it-club

[
![GitHub Pages](https://img.shields.io/badge/demo-online-blue.svg)
](https://code4fukui.github.io/music-kanazawa-it-club/)

「金沢IT部活」プレイリスト用のウェブベースの音楽プレイヤーで、Sunoによって生成されたAI音楽を再生します。このプロジェクトはスタンドアロンのシングルページアプリケーションであり、GitHub Pagesなどのサービスで簡単にホスティングできます。

プレイヤーは、現在のトラックのアルバムアートに合わせて動的に更新されるぼかし背景の上に、3カラムのレイアウトで表示されます：
1.  **プレイリスト:** アルバム内の全トラックのスクロール可能なリスト。
2.  **プレイヤー:** 現在のトラックのアルバムアート、オーディオコントロール、およびメタデータ。
3.  **歌詞:** 再生中の曲のフル歌詞。

## 機能

-   **動的背景:** アルバムアートを使用して、プレイヤーに没入感のあるぼかし背景を作成します。
-   **豊富なメタデータ表示:** 曲名、AI生成プロンプト、タグ、およびフル歌詞を表示します。
-   **高度な再生コントロール:** `riffleShuffle`や「お気に入り（liked）」の曲を優先する機能など、カスタムのソートロジックを実装しています。
-   **Media Session APIの統合:** ロック画面、通知センター、または接続されたハードウェア（ヘッドホンやカーステレオなど）といったOSのUIから、再生コントロール（再生/一時停止、次へ/前へ、シーク）が可能です。
-   **スタンドアロンかつシンプル:** サーバーサイドのバックエンドを必要とせず、すべてブラウザ上で動作します。

## 自分のSunoプレイリストでの利用方法

このプレイヤーは、安定したホスティングのためにローカルの音声ファイルと画像ファイルを使用するように設定されています。ご自身のSunoプレイリストに適用するには、以下の手順を実行してください：

1.  **プレイリストデータの取得:** Sunoのプレイリストページで、ブラウザの開発者ツールを使用してプレイリストのJSONデータを見つけ、保存します（例として `playlist_org.json` を参照してください）。
2.  **アセットのダウンロード:** JSONデータにリストされているすべての `.mp3` 音声ファイルと `.jpeg` 画像ファイルをダウンロードします。
3.  **ファイルの整理:** ダウンロードしたファイルを `audio/`、`image/`、および `image_large/` ディレクトリに配置します。
4.  **プレイリストの更新:** `playlist.json` ファイルを作成します。その中で、各トラックの `audio_url`、`image_url`、および `image_large_url` のパスを変更し、ローカルファイルを参照するようにします（例: `"audio/your-song-id.mp3"`）。
5.  **デプロイ:** 変更をコミットし、ウェブサーバーまたはGitHub Pagesを使用してリポジトリをホストします。
